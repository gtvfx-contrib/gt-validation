# Production Mock Pack Demo Manifest

## Session Use

`production_mock_pack` is intentionally larger and noisier than the primary demo
fixtures. It is suitable for scale, reporting, and production-noise discussions,
not first-pass live coding.

## Recommended Use

- Use after students already understand the validator behavior on
  `filesystem_pack`.
- Run as a comparison target when discussing runtime, output filtering, or
  prioritization.
- Avoid using it as the default target for Sessions 03-06.

## Instructor Rule

If the lesson is not explicitly about scale, do not use this pack live. It can
turn a clear demo into output triage.
